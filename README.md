
# TATAI - 终端访问控制系统

TATAI 是一个轻量级的应用进程管理平台，提供美观的 Web 界面用于统一管理各类服务进程。无论您是运行 Docker 容器、Java JAR 包、Nginx 服务还是自定义脚本，TATAI 都能让您通过一个控制台完成启停、监控、日志查看等操作。

## 核心特性

### 📦 多类型应用管理
- **Docker 容器**：管理运行中的容器，自动识别映射端口
- **Java JAR 应用**：支持多 JDK 版本切换，可配置 JVM 内存参数
- **Nginx 服务**：解析配置文件获取监听端口，支持重载配置
- **Other 自定义进程**：任意命令行程序，支持启停检测命令

### 🛡️ 智能进程守护
- 自动检测应用崩溃并重启
- 可配置最大重试次数和统计时间窗口
- 指数退避策略，防止频繁重启导致资源耗尽
- 重启历史记录，便于排查问题

### 📡 实时日志流
- WebSocket 长连接推送，毫秒级延迟
- 支持历史日志回溯（最近 50 行）
- 按应用独立日志文件存储
- 自动滚动/手动滚动自由切换

### 📊 系统资源监控
- CPU / 内存 / 磁盘使用率实时展示
- TOP 5 进程排行（CPU / 内存占用）
- 磁盘清理建议：智能识别大文件并支持一键清空

### 🔔 Webhook 告警通知
- 应用崩溃时自动发送告警
- 支持自定义 Webhook 地址
- 可配置触发事件类型
- 提供测试功能验证连通性

### 👥 用户权限管理
- 多用户支持，角色分离（admin / operator）
- 管理员可创建/禁用用户
- 管理员可重置普通用户密码
- 普通用户可修改自己的密码

### 🎨 现代化界面
- 深色主题设计，护眼且专业
- 响应式布局，支持多种屏幕尺寸
- 应用列表支持类型/状态筛选
- 应用排序可拖拽调整（开发中）

## 快速体验

### 部署方式

1. **下载最新版本**
```bash
git clone https://github.com/your-repo/tatai.git
cd tatai
```

2. **启动服务**
```bash
# 直接运行
./tatai

# 或使用 go run
go run main.go
```

3. **访问控制台**
   打开浏览器访问 `http://localhost:3000`

### 默认凭证

| 用户名 | 密码     | 角色 |
|--------|--------|------|
| admin | 123456 | 管理员 |

> 首次登录后请及时修改密码

## 应用配置示例

### Java JAR 应用
```json
{
  "type": "jar",
  "appName": "user-service",
  "jdkKey": "jdk17",
  "jarPath": "/opt/apps/user-service.jar",
  "memory": "512m",
  "ports": "[8080, 8081]"
}
```

### 自定义进程
```json
{
  "type": "other",
  "startCmd": "./server --port 3000",
  "stopCmd": "pkill -f server",
  "checkCmd": "pgrep -f server",
  "ports": "[3000]"
}
```

### Docker 容器
```json
{
  "type": "docker",
  "appName": "nginx-proxy",
  "cmd": "docker run -d -p 80:80 nginx"
}
```

## 守护策略说明

当启用应用守护后，TATAI 会持续检测应用运行状态：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 最大重试次数 | 窗口期内允许的最大重启次数 | 5 次 |
| 统计窗口 | 重置计数器的时间窗口 | 60 秒 |
| 退避基准 | 重启间隔递增基数 | 2 秒 |
| 最大退避 | 单次最大等待时间 | 60 秒 |

重启延迟计算公式：`delay = min(基准 × 2^(次数-1), 最大退避)`

示例：
- 第 1 次崩溃：等待 2 秒
- 第 2 次崩溃：等待 4 秒
- 第 3 次崩溃：等待 8 秒
- 第 4 次崩溃：等待 16 秒
- 第 5 次崩溃：等待 32 秒

## 端口匹配检测

对于 JAR 和 Other 类型应用，TATAI 会：

1. 读取用户配置的期望端口（JSON 数组格式）
2. 通过 `lsof -p {PID}` 获取进程实际监听端口
3. 在界面上对比展示匹配状态：
    - ✅ **matched**：期望端口与实际端口一致
    - ⚠️ **mismatch**：端口不一致，点击可查看详情
    - ❓ **unknown**：无法获取实际端口

## 存储与数据

- **数据库**：SQLite，文件位于 `./data/tatai.db`
- **日志文件**：`./logs/{app_name}_{app_id}.log`
- **配置文件**：`./config.yaml`（可选，支持环境变量覆盖）
